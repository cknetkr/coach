## 1. 서비스 개요

### 1.1 핵심 문제 정의

현행 문학 수행평가(비평문 작성)는 학생이 혼자 해결하기 어려운 **3중 난이도 구조**를 갖는다.

1. **내용 이해 난이도** — 작품의 문학적·사회적 맥락 분석
2. **글쓰기 구조 난이도** — 도입/전개1/전개2/전개3/정리 5문단 논리 설계
3. **평가 요건 충족 난이도** — AI 활용 증거, 정보 교차 검증, 참고자료 2건 이상, 1500±50자 등 루브릭 체크리스트

기존 AI 도구(ChatGPT, Gemini 등)는 **글을 대신 써주는 것**에 그치며, 평가 루브릭과의 1:1 매핑 및 학생 스스로의 사고 과정 증거 생성을 지원하지 않는다.

### 1.2 솔루션 방향

> **"글을 써주는 AI가 아닌, 만점을 받게 하는 AI 코치"**

- 실물 수행평가지(1·2·3) 폼을 100% 미러링하여 항목 누락 원천 차단
- 학생의 사고 과정 자체를 **수행평가 제출 증거**로 변환
- 수행평가3: 작성 완료 후 **시험장 대비 내재화 훈련**까지 연결

### 1.3 한 권의 책으로 전체 관통

수행평가1·2·3은 **동일한 소설 한 권**을 기반으로 연결된다.

```
[소설 선택] → [수행평가1: 독서주제탐구] → [수행평가2: 비평문 계획] → [수행평가3: 비평문 원고 작성]
     ↑                                                                              ↓
     └────────────── 같은 책 제목·지은이·사회 문제 주제가 전 단계 공유 ──────────────┘
```

## 2. 실물 수행평가지 완전 분석

### 2.1 수행평가1 — 독서주제탐구 (전체 항목)

| # | 항목명 | 내용 | v1 기획 반영 여부 |
|---|--------|------|-----------------|
| 1 | 작품 선정 이유 | 구체적으로, 어떤 점이 흥미로웠는가 | ✅ 있음 |
| 2 | 인물 및 배경 분석 | 주요 인물, 인물 특징, 시대·사회적 배경 | ✅ 있음 |
| 3 | 간단한 줄거리 | 주요 사건 중심, 책 전체 내용이 드러나게 | ❌ **누락** |
| 4 | 책이 나에게 준 생각과 태도의 변화 | 책 읽고 난 후 생각의 변화 | ❌ **누락** |
| 5 | 책의 주제와 연결되는 나의 경험과 사회 현실 | 작품 내용이 나의 경험·사회 문제와 연결되는 지점 | ❌ **누락** |
| 6 | 인상적인 문구·그 이유 (2분) | 문구(page), 이유 — 2개 항목, 내용 겹치지 않게 | ⚠️ 부분 반영 |

### 2.2 수행평가2 — 비평문 계획하기 (전체 항목)

| # | 항목명 | 세부 구조 | v1 기획 반영 여부 |
|---|--------|----------|-----------------|
| 1 | 작품의 주제와 관련된 사회 문제 제시 | 대표적 한 가지 | ✅ 있음 |
| 2 | 사회 문제 분석 | 발생 원인, 참고자료 2가지 이상 | ✅ 있음 |
| 3 | 해결 방안 탐색 | 구체적 방안, 참고자료 2가지 이상 | ✅ 있음 |
| 4 | AI 활용 아이디어 정리 | **질문+아이디어 쌍 × 4세트** | ❌ **구조 오류** (3건 단순 카운트로 설계됨) |
| 5 | 정보 검증 과정 | **번호 목록 3칸** (1. 2. 3.) | ❌ **구조 오류** (자유 텍스트로 설계됨) |
| 6 | 비평문 구조 계획 | 도입/전개1·2·3/정리 칸 + 보조자료 활용 방안 4칸 | ❌ **완전 누락** |

### 2.3 수행평가3 — 비평문 원고 작성 (조건 전체)

| # | 조건 | 검증 방식 |
|---|------|----------|
| 1 | 제목과 관련된 서평 주제를 담을 것 (내용 유사하지 않도록) | AI 검토 |
| 2 | [도입]-[전개1]-[전개2]-[전개3]-[정리] 5개 문단, **반드시 문단으로 나누어 서술** | 자동 카운트 |
| 3 | 통일성·응집성을 갖춘 완결된 글 | AI 검토 |
| 4 | 청색 또는 흑색 볼펜 사용, 수정테이프 사용 가능, 글씨를 또박또박 | 내재화 훈련 안내 |
| 5 | 올바른 맞춤법, 띄어쓰기, **본량 1500±50자** | 자동 카운트 |
| 6 | **다른 자료나 메모 등은 참고할 수 없습니다** | ⚠️ 내재화 훈련 필수 |

## 3. 서비스 구조 설계 (v2 전면 보강)

### 3.1 3단계 코칭 플로우

```
[책 선택 온보딩]
       ↓
[STEP 1] 독서주제탐구 코치     →  수행평가1 완성 (6개 항목 100% 미러링)
       ↓ (데이터 자동 연동)
[STEP 2] 비평문 계획 코치      →  수행평가2 완성 (6개 항목 100% 미러링)
       ↓ (데이터 자동 연동)
[STEP 3] 비평문 원고 작성 코치 →  수행평가3 완성 (1500±50자 원고지)
       ↓
[STEP 4] 시험장 대비 내재화 훈련 →  손글씨 시험 만점 준비 ← v2 신규
```

### 3.2 핵심 기능 모듈 (v2 전면 보강)

#### Module A — 루브릭 매니저 (Rubric Manager)
- 수행평가1·2·3 조건을 JSON 데이터로 관리
- 각 입력 항목과 루브릭 점수 항목 1:1 매핑
- 실시간 충족 여부 시각화 (체크리스트 UI)
- **미충족 항목 자동 감지 + 유도 질문 생성**

#### Module B — AI 프롬프트 코치 (Prompt Coach)
- 학생이 AI에게 질문하는 법을 가이드
- **질문+아이디어 쌍 4세트** 구조로 UI 설계 (실물 양식 100% 일치)
- AI가 질문을 대신 만드는 것이 아닌, 학생 스스로 질문 구성 유도

#### Module C — 정보 검증 어시스턴트 (Fact-Check Assistant)
- AI 답변의 통계·수치 생성 후, 검증 기관 링크 자동 제공
- **번호 목록 3칸 구조**로 입력 폼 설계 (실물 양식 100% 일치)
- 통계청 KOSIS, 국토연구원, 국회입법조사처 등 바로가기 제공

#### Module D — 5문단 구조 빌더 (Structure Builder)
- 수행평가2의 **구조 계획 칸** + 수행평가3의 **원고 작성 칸** 연동
- 도입/전개1·2·3/정리 각 칸 가이드 프롬프트
- 보조자료 활용 방안 4칸 (수행평가2 실물 양식 반영)
- 문단 간 논리 연결성(응집성) 자동 검토
- 통일성 점검: 전체 주제와 이탈 여부 AI 분석

#### Module E — 원고지 변환기 (Manuscript Converter)
- 완성 비평문을 1500±50자 자동 조정
- 수행평가3 원고지 형식으로 시각화 출력
- **단, 출력·인쇄 목적이 아닌 내재화 훈련 전 최종 확인용**

#### Module F — 내재화 훈련 코치 (Memorization Coach) ← **v2 신규**
- 수행평가3 시험형 대비 전용 모드
- 훈련 방식:
  1. **문단 요약 카드**: 각 문단을 핵심 키워드 3개로 압축
  2. **빈칸 채우기**: 핵심 문장에서 키워드를 빈칸으로 제시
  3. **재현 테스트**: 카드 없이 문단을 직접 타이핑하여 원본과 비교
  4. **글자수 시뮬레이션**: 손으로 쓸 때 예상 시간 안내 (분당 평균 30자 기준)

## 5. 기술 스택

### 5.1 프론트엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | React 19 + Vite | SPA 빠른 구동, 컴포넌트 재사용 |
| 스타일링 | TailwindCSS | 반응형 빠른 구현 |
| 상태관리 | Zustand | 단계 간 데이터 연동 (책 제목·지은이·사회 문제 공유) |
| 에디터 | TipTap | 리치 텍스트 + 글자수 실시간 카운팅 |
| 폼 관리 | React Hook Form | 수행평가 양식 미러링 폼 관리 |

### 5.2 백엔드 / AI

| 항목 | 선택 | 이유 |
|------|------|------|
| AI API | Gemini 1.5 Flash | 비용 효율 + 한국어 성능 |
| DB | Firebase Firestore | 실시간 저장, 세션 복구 |
| 호스팅 | Firebase Hosting | 무료 티어로 MVP 운영 가능 |
| 인증 | Firebase Auth (익명) | 회원가입 없이 즉시 사용 |

### 5.3 Gemini API 프롬프트 전략

```
시스템 프롬프트 설계 원칙:
1. 역할 설정: "너는 고2 문학 수행평가 전문 코치야.
              수원하이텍고등학교 2026학년도 수행평가 기준을 따른다."
2. 제약 설정: "직접 비평문을 써주지 마.
              학생 스스로 쓸 수 있도록 방향만 제시해."
3. 루브릭 주입: 평가 조건 전체를 프롬프트에 상시 포함
4. 단계 컨텍스트: 이전 단계(수행평가1·2) 데이터를 다음 단계에 자동 주입
5. 출력 형식: JSON 구조화 응답으로 UI 자동 매핑
```

### 5.4 비용 최적화 (배치 호출 전략)

- STEP별 AI 호출을 **묶음 처리** (단계 완료 시 1회 배치)
- 실시간 타이핑마다 API 호출 ❌ → 디바운싱 3초 적용
- 예상 월 API 비용 (100명 기준): 약 $2~5 수준

## 7. 콘텐츠 설계 — 『난장이가 쏘아올린 작은 공』 케이스

> 이 케이스는 서비스 초기 **시드 콘텐츠**로 탑재한다.

### 7.1 STEP 1 내장 가이드 콘텐츠 (v2 보강)

```
[작품 선정 이유]
❌ "재미있어서" → 감점 요인
✅ "현재 주거 불평등 문제와 연결할 수 있어서"
✅ "비판적 글쓰기에 적합한 사회적 모순을 담고 있어서"

[인물·배경 분석]
주요 인물: 이름 + 상징하는 계층/역할까지 기술
배경: 시간적(1970년대 산업화) + 공간적(낙원구 행복동) + 사회적(철거·재개발) 3요소

[간단한 줄거리] ← v2 신규
주요 사건 흐름: 철거 통보 → 난장이 가족의 저항 → 비극적 결말
책 전체 내용이 드러나도록 3~5문장으로 서술

[생각과 태도의 변화] ← v2 신규
"읽기 전: 빈곤은 개인의 노력 부족 문제라고 생각했다"
→ "읽은 후: 구조적 불평등이 개인의 삶을 어떻게 짓누르는지 인식하게 됐다"

[경험·사회 현실 연결] ← v2 신규
개인 경험 예시: 주거비 부담 / 재개발로 인한 이웃 이주
사회 문제 연결: 2024년 전세 사기 문제 / 청년 주거 불안

[인상적 문구 2분]
1분: "우리 아버지는 난장이였다." (p.12) → 사회적 약자를 상징
2분: "공장 굴뚝 위에서 뛰어내리셨어." (p.208) → 절망의 극단적 표현
⚠️ 두 문구 내용 겹치지 않도록 주제 다양화 안내
```

### 7.2 STEP 2 내장 AI 질문 템플릿 (4세트)

```
세트1
  질문: "난장이가 쏘아올린 작은 공의 핵심 갈등 구조와
        2026년 현재 주거 불평등 문제의 공통점은?"
  아이디어: 소설 속 철거 문제 = 현대 재개발 갈등 구조와 병치

세트2
  질문: "주거 불평등의 현황을 보여주는 신뢰할 수 있는 통계는?"
  아이디어: KOSIS 청년 주거 실태 통계 활용

세트3
  질문: "주거 불평등 해결을 위한 법적·제도적 대안을 구체적 사례와 함께 제시해줘"
  아이디어: 공공임대주택 확대 정책 / 전세 사기 방지법 사례

세트4
  질문: "산업화 시대 노동자 계층의 삶과 현재 저소득층의 삶을 비교해줘"
  아이디어: 시대적 유사성으로 비평문 전개2 강화
```

### 7.3 STEP 3 5문단 구조 템플릿

```
[도입] — 핵심 전략
→ 작품의 인상적 장면/문구 인용으로 시작
→ 현재 주거 불평등 문제와 연결 고리 선언

[전개1] — 원인 분석
→ 소설 속 근거 (철거 통보, 난장이 가족의 처지)
→ 사회 구조적 원인 분석 (1970년대 개발 독재)

[전개2] — 현재적 양상
→ 현대 주거 불평등 문제와 병치
→ 통계·데이터 근거 제시 (KOSIS 청년 주거 실태)

[전개3] — 해결방안
→ 법적/제도적 대안 (공공임대 확대, 전세 사기 방지법)
→ 사회적 인식 변화 촉구

[정리] — 마무리
→ 작품의 메시지 재환기 (난장이의 공 = 희망)
→ 미래 지향적 결론
```
